# Contributing to auto-story-tools

## Repository rule

Before adding anything, preserve this boundary:

> **Works with plain API keys by default; supports gateway/proxy transports as optional integrations.**

That means the core packages stay generic. If something only makes sense for one environment, gateway, OAuth flow, or local runtime, put it under `integrations/` or document it as an optional setup.

## Adding a new adapter

The project is designed so new output formats (novel, serial, interactive fiction) can be added as separate packages under `src/`.

### Structure

Each adapter lives in its own package:

```
src/
├── auto_outline/       # Foundation engine (generates the story bible)
├── auto_screenplay/    # Screenplay adapter
├── auto_novel/         # Your new adapter
└── ...
```

### What an adapter does

1. **Reads** the story bible files generated by `auto_outline` (world.md, characters.md, outline.md, etc.)
2. **Generates** format-specific output (Fountain for screenplays, chapter markdown for novels, etc.)
3. **Evaluates** the output using format-specific criteria (page count for screenplays, word count for novels, etc.)
4. **Iterates** using the same keep/discard logic as the foundation loop

### Shared resources

- `src/auto_outline/provider.py` — LLM provider abstraction. Use this for API calls.
- `src/auto_outline/state.py` — State management. Extend the state schema for your adapter's needs.
- `src/auto_outline/evaluation/mechanical.py` — Slop detection. Works on any text.

### Development workflow

```bash
# Install dev dependencies
uv sync --extra dev

# Run tests
pytest

# Lint before committing
ruff check src/ tests/
ruff format src/ tests/

# Type check
mypy src/
```

### Code style

- Python 3.12+
- Ruff for linting and formatting
- No AI slop in docstrings or comments (practice what we preach)
- Tests for any evaluation logic

### Pull requests

- One feature per PR
- Include tests for new evaluation dimensions or generators
- Update `docs/` if you change the architecture
- Reference the relevant issue in the PR title
